# CLAUDE.md - DOUGLAS 개발 규칙

## 핵심 규칙
**모든 작업은 문서 업데이트와 반드시 세트로 수행한다.**

## 빌드
- 코드 변경 후 반드시 `swift build -c release` 확인
- 빌드 실패 시 커밋 금지

## 문서
- ARCHITECTURE.md: 현재 코드 상태 반영 (구조 변경 시 필수 업데이트)
- DEV_GUIDE.md: 개발 규칙/관례 변경 시 업데이트

## Git
- 커밋 형식: `[DG] <type>: <설명>`
- type: feat, fix, refactor, style, docs
- force push 금지
- 빌드 통과 후에만 커밋

## 프로젝트 정보
- Swift 5.9, macOS 14+ (SPM)
- MVVM 아키텍처
- 빌드: `swift build -c release`
- 배포: `scripts/build-app.sh`

## 워크플로우 플로우 (절대 규칙)

**아래 9개 케이스의 플로우는 구현의 근간이다. 코드 수정 시 반드시 이 플로우가 깨지지 않는지 확인한다.**
**구현 전 ARCHITECTURE.md의 워크플로우 섹션을 반드시 읽고 참고한다.**

### Case 1: 단순 질의응답
프롬프트 입력 → 질의응답 판단 → 방 생성 → 에이전트 매칭 후 확인 요청(거부 시 직접 선택) → 에이전트 응답 완료

### Case 2: 토론 (1개 에이전트)
프롬프트 입력 → 토론 판단 → 방 생성 → 에이전트 매칭 후 확인 요청(거부 시 직접 선택) → 에이전트 응답 완료

### Case 3: 토론 (복수 에이전트)
프롬프트 입력 → 토론 판단 → 방 생성 → 에이전트 매칭 후 확인 요청(거부 시 직접 선택) → 에이전트끼리 토론(라운드마다 사용자 의견 가능) → 토론 종료 → DOUGLAS가 결론 도출 후 완료

### Case 4: 구현 (1개 에이전트)
프롬프트 입력 → 구현 판단 → 방 생성 → 에이전트 매칭 후 확인 요청(거부 시 직접 선택)
→ DOUGLAS가 실행 계획 도출 → 사용자 승인 or 요건 추가(승인될 때까지 무한 반복)
→ 실행 중 사용자가 실시간으로 구현 방향 추가 가능(해당 단계에 반영)
→ **마지막 단계 직전 반드시 승인 요청** → 승인 시 완료, 거부 시 지정 단계로 롤백 후 재수행

### Case 5: 구현 (복수 에이전트)
프롬프트 입력 → 구현 판단 → 방 생성 → 에이전트 매칭 후 확인 요청(거부 시 직접 선택)
→ 에이전트끼리 토론(라운드마다 사용자 의견 가능)
→ 토론 종료 후 DOUGLAS가 실행 계획 도출 → 사용자 승인 or 요건 추가(승인될 때까지 무한 반복)
→ 실행 중 사용자가 실시간으로 구현 방향 추가 가능(해당 단계에 반영)
→ **마지막 단계 직전 반드시 승인 요청** → 승인 시 완료, 거부 시 지정 단계로 롤백 후 재수행

### Case 6: 방 생성
Case 1~5와 동일하되, 에이전트 매칭 단계만 없음.

### Case 7: 파일 생성
- 7-1. 진행 중/완료 후 "문서 만들어줘" → 대화 요약 후 문서 생성. 방에 내용 표시 불필요, "완료되었습니다." + 경로만 표시.
- 7-2. 처음부터 "~~에 대해 문서로 만들어줘" → 웹서칭 등 조사 후 문서 생성. 동일하게 담백하게 완료 메시지 + 경로만.

### Case 8: 후속 처리
모든 케이스 완료 후 후속 프롬프트 가능. 입력을 기반으로 질의응답/토론/구현/파일생성 구분 후 해당 플로우 수행.

### Case 9: 요건 불명확
요건이 불명확하면(URL만, 이미지만 등) DOUGLAS가 재차 질문하여 명확화.
