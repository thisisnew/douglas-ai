# CLAUDE.md - DOUGLAS 개발 규칙

## 핵심 규칙
**모든 작업은 문서 업데이트와 반드시 세트로 수행한다.**

## 빌드
- 코드 변경 후 반드시 `swift build -c release` 확인
- 빌드 실패 시 커밋 금지

## 문서
- ARCHITECTURE.md: 현재 코드 상태 반영 (구조 변경 시 필수 업데이트)
- DEV_GUIDE.md: 개발 규칙/관례 변경 시 업데이트

## Git
- 커밋 형식: `[DG] <type>: <설명>`
- type: feat, fix, refactor, style, docs
- force push 금지
- 빌드 통과 후에만 커밋

## 프로젝트 정보
- Swift 5.9, macOS 14+ (SPM)
- MVVM 아키텍처
- 빌드: `swift build -c release`
- 배포: `scripts/build-app.sh`

## 워크플로우 (절대 규칙)

**워크플로우 명세는 `WORKFLOW_SPEC.md`가 정본이다. 코드 수정 시 반드시 이 문서를 읽고 플로우가 깨지지 않는지 확인한다.**
**구현 전 ARCHITECTURE.md의 워크플로우 섹션도 함께 참고한다.**

### 필수 준수 사항
- 상태 머신 기반 동작 (케이스 나열이 아님)
- 불명확하면 실행보다 재질문 우선
- 고비용 작업은 계획 승인 필수
- 되돌릴 수 없는 작업은 직전 승인 필수
- 후속 입력은 기존 맥락 위에서 해석

### 공식 케이스 (A~H)
- **A. 질의응답**: 단일 에이전트/DOUGLAS 직접 응답
- **B. 단일 에이전트 토론**: 심화 질의응답 수준의 단독 검토
- **C. 복수 에이전트 토론**: 라운드 기반 토론 → 결론/대안/쟁점 정리
- **D. 단일 에이전트 구현**: 계획 → 승인 → 실행 → 최종 승인
- **E. 복수 에이전트 구현**: 사전 토론 → 계획 → 승인 → 실행 → 최종 승인
- **F. 문서 생성**: 문서 전문가가 최종 책임, "완료되었습니다." + 경로만 표시
- **G. 후속처리**: 완료 후 같은 방에서 다른 모드로 확장 가능
- **H. 요건 불명확**: 재질문 후 명확해질 때까지 실행 보류

상세 흐름, 상태 전이, 승인 모델, 예외 처리는 `WORKFLOW_SPEC.md` 참조.
