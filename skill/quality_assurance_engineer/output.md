# Output Files

## 생성 폴더

최종 결과물은 아래 폴더 기준으로 정리한다.

`[기능명]_테스트케이스/`

## 생성 파일 목록

### 1. `[기능명]_테스트케이스.csv`
용도:
- TestRail 업로드용 UI/UX 테스트케이스 파일

포함 내용:
- 정상 케이스
- 오류 케이스
- 엣지 케이스
- 핵심 기능 검증
- 입력 검증
- 상태 변경
- 출력/다운로드/전송/일괄 처리 등 주요 액션

---

### 2. `[기능명]_보안테스트케이스.csv`
용도:
- TestRail 업로드용 보안 테스트케이스 파일

포함 내용:
- SQL Injection
- XSS
- 인증 우회
- 권한 우회
- URL 파라미터 조작
- 타 조직/타 사용자 데이터 접근 시도

---

### 3. `[기능명]_리스크분석.md`
용도:
- 기능 위험 요소 분석

포함 내용:
- 위험 항목
- 발생 가능성
- 영향도
- 우선순위
- 완화 전략

---

### 4. `[기능명]_질문.md`
용도:
- 불명확한 요구사항, 정책 미정 사항, 확인 필요 항목 정리

포함 내용:
- 요청 원문 URL 또는 문서 정보
- 우선순위별 질문
- 테스트 설계에 직접 영향 주는 항목 우선 정리

---

### 5. `[기능명]_체크리스트.md`
용도:
- 커버리지 검증용 최종 체크리스트

포함 내용:
- 기능별 커버 여부
- 보안 항목 포함 여부
- 핵심 액션 포함 여부
- 입력 오류/예외 케이스 포함 여부
- 성능/네트워크 예외 포함 여부

## 파일명 규칙

파일명은 반드시 기능명을 접두로 사용한다.

예시:
- 로그인_테스트케이스.csv
- 로그인_보안테스트케이스.csv
- 로그인_리스크분석.md
- 로그인_질문.md
- 로그인_체크리스트.md

## 산출물 작성 순서

권장 순서는 다음과 같다.

1. 테스트케이스.csv
2. 보안테스트케이스.csv
3. 질문.md
4. 리스크분석.md
5. 체크리스트.md

질문 문서는 테스트 설계 중간에 병행 작성해도 된다.
