# Overview

## 목적

이 스킬은 요구사항 문서를 분석하여 TestRail에 업로드 가능한 테스트 케이스 파일과 보조 문서를 생성한다.

생성 대상은 다음과 같다.

- UI/UX 테스트케이스 CSV
- 보안 테스트케이스 CSV
- 리스크 분석 문서
- 요구사항 확인 질문 문서
- 커버리지 체크리스트 문서

## 핵심 원칙

### 1. 정확성
불명확한 요구사항은 추측하지 않는다.  
확정할 수 없는 내용은 `질문.md`에 기록한다.

### 2. 실행 가능성
누구나 동일하게 수행할 수 있도록 테스트 데이터, 접근 경로, 기대 결과를 구체적으로 작성한다.

### 3. 완전한 커버리지
Happy Path뿐 아니라 Edge Case, 입력 오류, 인증/인가, 보안 취약점, 예외 상황까지 포함한다.

### 4. 파일 분리
UI/UX 테스트케이스와 보안 테스트케이스는 반드시 별도 CSV 파일로 분리한다.

## 입력 문서 범위

다음 유형의 문서를 입력으로 받을 수 있다.

- Figma
- Axure
- Jira
- Confluence
- 일반 웹 URL
- 스크린샷
- PPT
- PDF

## 기본 동작 원칙

1. 먼저 입력 문서에서 기능 범위, UI 요소, 사용자 흐름, 제약사항을 파악한다.
2. 다음으로 기능 단위의 Section 구조를 설계한다.
3. 이후 UI/UX 테스트케이스와 보안 테스트케이스를 별도 작성한다.
4. 요구사항이 불명확하면 질문 문서를 만든다.
5. 마지막으로 리스크 분석과 체크리스트를 작성한다.

## 결과물 품질 기준

산출물은 아래 기준을 만족해야 한다.

- 테스트 제목만 보고도 목적이 드러날 것
- Preconditions에 권한, 접근 경로, 데이터가 포함될 것
- Steps는 한 단계당 하나의 행동만 포함할 것
- Expected Result는 검증 가능한 형태일 것
- 보안 테스트는 최소 3건 이상 포함할 것
- 누락된 요구사항은 질문 문서에 별도 정리할 것
